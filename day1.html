What is the difference between var, let, and const in JavaScript?
JavaScript Basic
📖 Answer
var is function-scoped and gets hoisted to the top of its function with a default value of undefined. It can be re-declared and updated.

let is block-scoped (lives only within {}), hoisted but NOT initialized (Temporal Dead Zone), can be updated but NOT re-declared in same scope.

const is also block-scoped, must be initialized at declaration, and cannot be reassigned. However, object properties or array elements inside a const can still be mutated.
🗣 Explanation (Hinglish)
// Hinglish explanation Yaar, basically teen cheezein hain scope ke baare mein:

var — purana zamana, function ke bahar bhi visible ho jaata hai agar function ke andar likha ho — isliye bugs aate the. Hoisting hoti hai matlab variable ka naam upar chala jaata hai par value undefined rehti hai.

let — modern way. Sirf apne curly braces {} ke andar valid hai. TDZ (Temporal Dead Zone) matlab declare se pehle access karo toh error milegi.

const — constant matlab ek baar assign kiya toh dobara assign nahi kar sakte. Lekin agar object ya array hai, toh uske andar ke values change ho sakte hain. Ye common interview trick hai!
💻 Code Example
javascript 
// ---- var example ----
function testVar() {
  if (true) {
    var x = 10;
  }
  console.log(x); // 10 — var leaks out of if block!
}

// ---- let example ----
function testLet() {
  if (true) {
    let y = 20;
  }
  console.log(y); // ReferenceError: y is not defined
}

// ---- const example ----
const person = { name: "Rahul" };
person.name = "Priya"; // ✅ Allowed — mutating property
person = {};           // ❌ TypeError — can't reassign const

// ---- Hoisting trap (var) ----
console.log(a); // undefined (hoisted, not error)
var a = 5;

console.log(b); // ReferenceError — TDZ for let
let b = 5;

2
What is == vs === in JavaScript? Give examples where they differ.
JavaScript Basic Tricky
📖 Answer
== (Abstract Equality) performs type coercion before comparing — it converts both values to the same type, then compares.

=== (Strict Equality) compares both value AND type with no coercion. This is almost always what you want.
🗣 Explanation (Hinglish)
// Hinglish explanation Double equal (==) — "bhai type dekho mat, value dekho" wali approach. JavaScript pehle dono ko same type mein convert karta hai phir compare karta hai. Ye kabhi kabhi bahut weird results deta hai.

Triple equal (===) — strict comparison, type bhi same hona chahiye aur value bhi. Production code mein hamesha yahi use karo.

Interview tip: null == undefined is true, but null === undefined is false. Ye classic trick question hai!
💻 Code Example
javascript 
// Type coercion examples
console.log(0 == "");         // true  — both coerced to 0
console.log(0 === "");        // false — different types

console.log(false == 0);      // true  — false coerced to 0
console.log(false === 0);     // false

console.log(null == undefined);  // true  — special rule
console.log(null === undefined); // false — different types

console.log("1" == 1);         // true  — string coerced to number
console.log("1" === 1);        // false

// ⚠️ Always use === in real code

What is a closure in JavaScript? Why is it useful?
JavaScript Intermediate
📖 Answer
A closure is a function that remembers the variables from its outer scope even after the outer function has finished executing. In JavaScript, every function creates a closure over its surrounding lexical environment.

Closures are useful for: data privacy (encapsulation), factory functions, memoization, event handlers, and maintaining state in functional patterns.
🗣 Explanation (Hinglish)
// Hinglish explanation Sochlo ek backpack wala chhota function hai. Jab ye bahar jaata hai apni parent function ke, toh apne parent ke variables apne backpack mein leke jaata hai. Parent function khatam ho gayi, lekin wo variables abhi bhi accessible hain.

Real use case: counter banana jisme count private rahe — bahar se direct change na ho sake. Ya React mein event handlers jisme specific ID remember rahe. Closure = function + uska outer scope ka memory.
💻 Code Example
javascript 
// Basic closure
function makeCounter() {
  let count = 0; // private variable

  return {
    increment() { count++; },
    decrement() { count--; },
    getCount()  { return count; }
  };
}

const counter = makeCounter();
counter.increment();
counter.increment();
console.log(counter.getCount()); // 2
// count is not accessible directly — it's private!

// Closure in loop (classic interview trap)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 1000);
  // With let: prints 0, 1, 2  ✅
  // With var: prints 3, 3, 3  ❌ (classic bug!)
}

4
Explain event loop, call stack, and microtask queue in JavaScript.
JavaScript Intermediate
📖 Answer
JavaScript is single-threaded — it can only do one thing at a time. The call stack is where synchronous code executes, LIFO (Last In, First Out).

When async tasks (setTimeout, fetch) complete, their callbacks go to the task queue (macrotask queue). Promises go to the microtask queue, which has higher priority than the task queue.

The event loop continuously checks: if the call stack is empty → first drain the microtask queue completely → then take one macrotask. Repeat.
🗣 Explanation (Hinglish)
// Hinglish explanation JS ek hi kaam ek time mein karta hai — single-threaded. Call stack ek plate stack ki tarah hai — jo pehle andar aayi wo baad mein niklegi.

Jab Promise resolve hoti hai, uska callback microtask queue mein jaata hai. setTimeout ka callback macrotask queue mein. Event loop pehle microtasks khatam karta hai, phir ek macrotask lita hai, phir phir microtasks — ye cycle chalta rehta hai.

Isliye Promise.resolve().then() hamesha setTimeout(fn, 0) se pehle chalega — chahe dono ek saath likhe hon!
💻 Code Example
javascript 
console.log("1 — Sync");              // Call stack

setTimeout(() => {
  console.log("2 — setTimeout");      // Macrotask queue
}, 0);

Promise.resolve().then(() => {
  console.log("3 — Promise.then");    // Microtask queue
});

console.log("4 — Sync");              // Call stack

// Output order:
// 1 — Sync
// 4 — Sync
// 3 — Promise.then   ← microtask wins!
// 2 — setTimeout     ← macrotask last
Intermediate Event Loop — Mid-level JS interviews mein zaroor poochha jaata hai

What is this in JavaScript? How does it behave in arrow functions vs regular functions?
JavaScript Intermediate Tricky
📖 Answer
this refers to the object that is currently executing the function. Its value depends on how the function is called, not where it's defined (for regular functions).

Regular function: this is determined at call time — by the caller object, call/apply/bind, or new. In strict mode with no caller, it's undefined; in sloppy mode it's the global object.

Arrow function: does NOT have its own this. It inherits this from the surrounding lexical scope (where it was defined). This makes arrow functions great for callbacks.
🗣 Explanation (Hinglish)
// Hinglish explanation this ka matlab hai — "mujhe kis ne bulaya?" Regular function mein this caller decide karta hai. Agar koi object ne bulaya, toh this woh object hai. Agar directly bulaya, toh global ya undefined.

Arrow function apna this nahi banata — wo bahar se uthaa leta hai jahan likha gaya tha. Isliye React class components mein this.handleClick = this.handleClick.bind(this) ya arrow functions use karte hain — taaki this lost na ho event handler mein.
💻 Code Example
javascript 
const obj = {
  name: "Rahul",

  // Regular function — this = obj
  greetRegular: function() {
    console.log("Hello", this.name); // "Hello Rahul" ✅
  },

  // Arrow function — this = outer scope (window/global)
  greetArrow: () => {
    console.log("Hello", this.name); // "Hello undefined" ❌
  },

  // Arrow in setTimeout — this correctly captured
  delayedGreet() {
    setTimeout(() => {
      console.log("Delayed:", this.name); // "Rahul" ✅ arrow captures obj's this
    }, 1000);
  }
};

obj.greetRegular();
obj.greetArrow();
obj.delayedGreet();
Intermediate this — React + class interviews mein guarantee poochha jaata hai
6
What is the difference between null and undefined?
JavaScript Basic
📖 Answer
undefined means a variable has been declared but not assigned any value. JavaScript itself sets variables to undefined by default (e.g., function parameters not passed, uninitialized variables).

null is an intentional absence of value — a developer explicitly sets it to indicate "no value here." It's a deliberate assignment, not an accident.
🗣 Explanation (Hinglish)
// Hinglish explanation undefined = JavaScript bolta hai "bhai, tune declare kiya par kuch diya nahi." Ye JS khud set karta hai.

null = developer bolta hai "haan, main jaanta hoon — yahan deliberately kuch nahi hai." Ye tum set karte ho.

Ek aur interesting cheez: typeof null returns "object" — ye JavaScript ka ek purana bug hai! Interview mein jaroor poochhe jaate hain ye.
💻 Code Example
javascript 
let a;
console.log(a);            // undefined — declared but not assigned

let b = null;
console.log(b);            // null — explicitly set

// typeof
console.log(typeof a);     // "undefined"
console.log(typeof b);     // "object" ← famous JS bug!

// Equality
console.log(a == b);       // true  (both "nullish")
console.log(a === b);      // false (different types)

// Practical use
function getUser(id) {
  if (!id) return null;    // deliberate: "no user found"
}
let user = getUser(null);
console.log(user);          // null

What are React Hooks? Why were they introduced? Explain useState and useEffect.
React Intermediate
📖 Answer
React Hooks (introduced in React 16.8) are functions that let you use React state and lifecycle features in functional components — previously only available in class components.

useState: Returns a state variable and a setter. Re-renders the component when state changes.

useEffect: Runs side effects (data fetching, subscriptions, DOM manipulation) after render. The dependency array controls when it re-runs: empty [] = only on mount, with deps = when deps change, no array = every render.
🗣 Explanation (Hinglish)
// Hinglish explanation Pehle class components mein state aur lifecycle hota tha — this.state, componentDidMount etc. Bahut boilerplate tha aur this ki problem bhi.

Hooks aaye toh functional components mein bhi sab milne laga. useState se state banao, useEffect se side effects karo (API call, event listener add karna).

Dependency array samajhna zaroori hai — ye bataata hai useEffect kab chalega. Empty array = sirf ek baar mount pe. Array mein value = jab wo value badley. No array = har render pe — dangerous!
💻 Code Example
javascript 
import { useState, useEffect } from "react";

function UserProfile({ userId }) {
  // useState: [currentValue, setter]
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // useEffect: runs after render
  useEffect(() => {
    // Runs when userId changes
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });

    // Cleanup function (runs on unmount or before re-run)
    return () => {
      // cancel subscription, clear timers, etc.
    };
  }, [userId]); // dependency array

  if (loading) return <div>Loading...</div>;
  return <div>{user.name}</div>;
}

What is Virtual DOM in React and how does React reconciliation work?
React Intermediate
📖 Answer
The Virtual DOM (VDOM) is a lightweight JavaScript representation of the actual DOM. React maintains a VDOM in memory and syncs it with the real DOM through a process called reconciliation.

When state/props change: (1) React creates a new VDOM tree, (2) diffs it against the previous VDOM (using its diffing algorithm), (3) calculates the minimum DOM operations needed, and (4) batches and applies only those changes to the real DOM. This is far faster than manipulating the real DOM directly.
🗣 Explanation (Hinglish)
// Hinglish explanation Real DOM mein changes karna slow hota hai — browser ko reflow aur repaint karna padta hai. React ne ek jugaad socha — pehle memory mein ek "virtual" copy banao, wahan sab changes karo, phir compare karo real DOM se, aur sirf jo different hai wahi update karo.

Ye process reconciliation kehlaata hai. React ka diffing algo O(n) complexity mein kaam karta hai. Key prop lists mein isliye important hai — React ko batata hai ki konsa element kaunsa hai efficiently track karne ke liye.

React 18 mein Concurrent Mode aaya — ab React rendering ko interrupt aur prioritize kar sakta hai!
💻 Code Example
javascript 
// Why key is important in lists — helps reconciliation
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo => (
        // Without key: React repaints all items on change
        // With unique key: React knows exactly what changed
        <li key={todo.id}>{todo.text}</li>
      ))}
    </ul>
  );
}

// ❌ Bad — using index as key (breaks on reorder/delete)
todos.map((todo, index) => <li key={index}>{todo}</li>);

// ✅ Good — using stable unique id
todos.map(todo => <li key={todo.id}>{todo.text}</li>);

What is the difference between controlled and uncontrolled components in React?
React Basic
📖 Answer
Controlled Component: Form data is handled by React state. The input value is always driven by state — you control every keystroke via onChange and value props. React is the single source of truth.

Uncontrolled Component: Form data is handled by the DOM itself. You read values using ref when needed (e.g., on submit). The DOM is the source of truth.
🗣 Explanation (Hinglish)
// Hinglish explanation Controlled = React boss hai. Har keystroke pe state update hoti hai, value always React ke haath mein rehti hai. Ye approach better hai validation, conditional rendering, aur form management libraries (React Hook Form, Formik) ke liye.

Uncontrolled = DOM apna kaam khud karta hai, React sirf zaroorat padne par ref se value uthaa leta hai. Simple forms ya file inputs ke liye theek hai. File input hamesha uncontrolled hota hai — ye JS se set nahi ho sakta!
💻 Code Example
javascript 
import { useState, useRef } from "react";

// ✅ CONTROLLED — React controls value
function ControlledForm() {
  const [name, setName] = useState("");

  return (
    <input
      value={name}
      onChange={e => setName(e.target.value)}
      placeholder="Enter name"
    />
  );
}

// UNCONTROLLED — DOM controls value, ref reads it
function UncontrolledForm() {
  const nameRef = useRef(null);

  const handleSubmit = () => {
    console.log(nameRef.current.value); // read on demand
  };

  return (
    <>
      <input ref={nameRef} defaultValue="" />
      <button onClick={handleSubmit}>Submit</button>
    </>
  );
}

What is Promise in JS? Explain async/await and how it differs from .then()/.catch()?
JavaScript Intermediate
📖 Answer
A Promise represents an asynchronous operation that will complete in the future. It has three states: pending, fulfilled, or rejected.

.then()/.catch() is the original Promise API — chains callbacks. Can lead to "promise chaining" which is still better than callback hell.

async/await is syntactic sugar over Promises — it makes async code look and behave more like synchronous code. Under the hood, an async function always returns a Promise. await pauses execution until the Promise resolves (without blocking the main thread).
🗣 Explanation (Hinglish)
// Hinglish explanation Promise ek "I promise to give you data later" ki tarah hai. Teri API call gaye aur boli "bhai, main result baad mein dunga" — wo result Promise hai.

.then()/.catch() — purana tarika, chaining se kaam karo. Theek hai par nested ho jaata hai kabhi kabhi.

async/await — same kaam, par code seedha dikhta hai synchronous jaisa. Error handling ke liye try/catch use karo. Interview mein dono pata hone chahiye, real code mein async/await prefer karo.
💻 Code Example
javascript 
// --- Promise with .then/.catch ---
fetch("/api/user")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// --- Same with async/await ---
async function getUser() {
  try {
    const res = await fetch("/api/user");
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

// Parallel requests — use Promise.all
async function getAll() {
  const [users, posts] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/posts").then(r => r.json())
  ]);
  console.log(users, posts); // both ready at same time ✅
}
